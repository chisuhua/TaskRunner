# Phase 1.5 Stretch — Fix CudaScheduler dynamic_cast<CudaStub*> Hardcoding

> **For agentic workers:** Use `superpowers:subagent-driven-development` to implement. Steps use checkbox (`- [ ]`) syntax. This is a single-file, single-method refactor — each task is ~15-30 min.

**Goal:** Remove 5 `dynamic_cast<CudaStub*>(driver_)` sites in `src/test_fixture/cuda_scheduler.cpp` by routing CUDA Driver API calls through IGpuDriver virtual interface methods. Enable GpuDriverClient backend to serve `submitMemAlloc`/`submitMemFree`/`submitMemcpy*`/`submitLaunch`.

**Architecture:** CudaScheduler already uses `driver_->alloc_bo()`, `driver_->submit_memcpy()`, `driver_->submit_launch()` for other paths. This fix extends the same pattern to the 5 CUDA Driver API paths.

**Spec:** `docs/superpowers/specs/2026-07-02-phase1.5-cudastub-dynamic-cast-fix.md`

**Cross-repo:** No UsrLinuxEmu code change required. After fix, `libcuda_taskrunner.so` can route through `GpuDriverClient` → UsrLinuxEmu real GPU backend.

## Sub-plan C: Stub Mode Fence Handling

**Goal:** After removing `stub->is_stub_mode()` checks, ensure fence signaling works through IGpuDriver's `wait_fence` interface instead of CudaStub-specific immediate signal.

- [ ] **Step 1**: In each of the 5 fixed methods, after creating fence via `sync_mgr_.create_fence()`, add:
  ```cpp
  // Wait for driver fence completion (replaces immediate stub signal)
  uint32_t status = 0;
  driver_->wait_fence(fence_id, 0, &status);  // timeout=0 = infinite wait
  if (status != 1) {
      sync_mgr_.signal_fence(fence);  // fallback: signal anyway
  }
  ```
